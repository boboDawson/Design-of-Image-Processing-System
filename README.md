# 图像处理系统

## 项目简介
这是一个基于 Python 和 Tkinter 开发的图像处理桌面应用程序。系统集成了用户登录/注册、图像编辑、滤镜效果、操作历史记录以及日志追踪等功能，旨在提供一个功能丰富且易于扩展的图像处理平台。

## 功能特性
### 📝 用户系统
- 支持用户注册、登录、切换用户、退出登录
- 密码经过 SHA-256 加密存储，保障数据安全
- 通过文件记录登录状态，避免重复登录

### 🎨 图像编辑
- 基础操作：打开、保存图片（支持 JPEG、PNG、BMP 等格式）
- 几何变换：裁剪、旋转、翻转图像
- 水印与绘图：添加文字水印（自定义字体大小/颜色/粗细）、自由绘图（自定义画笔颜色/粗细）
- 滤镜效果：反相、黑白、风格化、素描、浮雕、复古、模糊、二值化等
- 色彩调整：色阶调整（亮度、对比度、RGB 通道独立调节）
- 形态学操作：边缘检测、腐蚀、膨胀

### 📜 操作与日志
- 操作历史：记录每一步操作，支持撤销和重置
- 日志记录：所有用户操作（登录、打开图片、应用效果等）均记录到日志文件和控制台
- 数据存储：使用 SQLite 存储用户信息、图片上传记录和处理日志

## 技术栈
| 类别         | 技术/库                          |

| 编程语言     | Python 3 (3.6+)                  |

## 安装与运行
### 环境要求
- Python 3.6 或更高版本
- 操作系统：Windows/macOS/Linux（兼容所有主流系统）

### 安装依赖
```bash
pip install opencv-python numpy Pillow
注：Tkinter、sqlite3、hashlib、logging 等为 Python 标准库，无需额外安装
运行方式
克隆或下载本项目到本地
进入项目根目录
启动程序：
bash
运行
# 主程序入口
python main.py

# 若主程序不可用，直接启动登录系统
python login_system.py
程序首次运行会自动创建 logs 文件夹和 SQLite 数据库文件（image_editor.db）
模块说明
表格
文件名	功能说明
main.py	程序入口，检查登录状态并启动对应界面
login_system.py	登录 / 注册界面，处理用户验证、注册逻辑、登录状态保存
image_editor.py	图像编辑器主窗口，包含所有图像处理功能、界面布局、历史记录管理
db_manager.py	数据库管理模块，封装 SQLite 连接和增删改查操作
login_interceptor.py	登录状态拦截器，通过 login_status.txt 实现自动登录
logger_config.py	日志配置模块，输出日志到控制台和按日期滚动的文件（logs/system_YYYYMMDD.log）
utils.py	工具函数模块，包含密码复杂度验证等通用功能
使用指南
首次使用
启动程序后进入登录界面，点击「注册」创建新账户
注册完成后输入用户名和密码，点击「登录」进入主界面
图像编辑流程
点击「打开图片」选择本地图片
使用左侧工具栏进行裁剪、添加文字、绘图等基础操作
点击右侧滤镜按钮应用各类特效
通过「应用」「取消」「重置所有」管理操作历史
用户管理
切换用户：点击顶部用户面板的「切换用户」，重新登录其他账户
退出登录：点击「退出登录」，系统会清除登录状态并返回登录界面
退出程序：直接关闭窗口即可，系统自动清理临时状态
数据库结构
users 表（用户信息）
表格
字段名	说明
user_id	用户唯一标识（主键）
username	用户名（唯一）
password	SHA-256 加密后的密码
email	用户邮箱
create_time	账户创建时间
last_login	最后登录时间
images 表（图片上传记录）
表格
字段名	说明
image_id	图片唯一标识（主键）
user_id	关联用户 ID
filename	图片文件名
filepath	图片存储路径
format	图片格式（JPG/PNG 等）
size	图片大小（字节）
upload_time	上传时间
process_logs 表（图像处理日志）
表格
字段名	说明
log_id	日志唯一标识（主键）
image_id	关联图片 ID
user_id	关联用户 ID
process_type	操作类型（滤镜 / 裁剪等）
parameters	操作参数（JSON 格式）
process_time	操作时间
注意事项
密码以 SHA-256 哈希值存储，无法找回，请务必牢记密码
确保项目目录有读写权限，否则无法创建日志文件夹和数据库文件
图像处理操作会实时记录到数据库，请勿随意删除 image_editor.db 文件
日志文件存储在 logs 目录下，按日期分割，便于问题排查
